# MindEase — Diretrizes de Game Design Cognitivo
Activation: Always On

<core_principle>
- Este projeto NÃO utiliza gamificação tradicional.
- Game design é usado apenas como suporte cognitivo e emocional.
</core_principle>

<forbidden_mechanics>
- Não usar pontuação, XP, ranking ou leaderboards.
- Não usar streaks, penalidades ou pressão por continuidade.
- Não usar feedback negativo ou comparação entre usuários.
- Não usar timers agressivos ou estímulos urgentes.
</forbidden_mechanics>

<allowed_mechanics>
- Micro-jogos opcionais e curtos (1–3 minutos).
- Interações destinadas a:
  - entrada em estado de foco
  - pausa cognitiva
  - transição entre tarefas
</allowed_mechanics>

<interaction_rules>
- Jogos nunca são obrigatórios.
- Usuário pode pular, pausar ou desativar qualquer interação lúdica.
- Não existe falha, derrota ou condição de sucesso explícita.
</interaction_rules>

<visual_guidelines>
- Movimento lento e previsível.
- Poucos elementos simultâneos na tela.
- Animações sempre opcionais.
- Evitar estímulos visuais intensos ou piscantes.
</visual_guidelines>

<language_guidelines>
- Evitar linguagem de cobrança ou performance.
- Preferir linguagem neutra e acolhedora.
- Exemplos permitidos:
  - “Você pode parar aqui”
  - “Isso já é suficiente”
  - “Quando estiver pronto”
</language_guidelines>
